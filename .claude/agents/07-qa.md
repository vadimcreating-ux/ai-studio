## Приоритетная задача
**Проверить и расширить покрытие тестами.** Текущее состояние:
- Smoke-тесты: `apps/tests/smoke.ts` — покрывают основные чат-операции
- E2E тесты: `apps/tests/e2e/` — Playwright против прода
- **НЕ покрыты smoke-тестами**: `/api/auth/register`, `/api/auth/me`, `/api/credits/balance`, `/api/credits/prices`, `/api/auth/logout`
- **НЕ покрыты**: большинство admin эндпоинтов (0/14)

## Тестовая инфраструктура

```bash
npm run test:smoke        # Запуск smoke-тестов
npm run test:e2e          # Запуск Playwright E2E
npm run cleanup:test-chats # Удалить мусор от упавших тестов
```

## Соглашения по тестам

### Smoke-тесты (`apps/tests/smoke.ts`)
```typescript
// Чаты создаются с заголовком [smoke-test] и удаляются в finally
// Проверять: поле существует, правильный тип, бизнес-логика корректна
assert(response.ok === true, "ожидали ok: true");
assert(typeof response.data.field === "number", "field должен быть number");
```

### E2E-тесты (`apps/tests/e2e/*.spec.ts`)
```typescript
// beforeAll/afterAll: запоминать существующие chatIds, удалять только созданные
// Не зависеть от порядка тестов
```

## При добавлении smoke-теста
1. Найти функцию `runXxxTests()` в `smoke.ts` или создать новую
2. Вызвать нужный эндпоинт через fetch с Bearer токеном
3. Проверить структуру ответа через assert
4. В `finally` удалить все созданные ресурсы
5. Добавить вызов в main()
